      1. Evergreen notes have some basic tenets to them:

Notes should be atomic

Create notes about one thing, but capture the entirety of that thing. The idea is to self-contain the note so it’s clear and concise, and able to be fully understood in any future context.



Notes should be concept-oriented

Orienting by concept makes it easier to gather context in the future and build associations between multiple notes. Especially between the notes that appear to be unrelated, but in a new, future context, become linked.



Notes should be densely linked

Connect notes bidirectionally to help spark new ideas based on related contexts



Notes should be emergent

Let structure emerge over time. Just take notes and start to connect them. This will lead to new ideas, new organization, and new output.



Notes should be titled like APIs

The title is an abstraction for the note itself. Titles provide a window into the content by being able to contain the idea itself while providing the ability to dive deeper into the idea when you want.
